# Инструменты агента Мадины (VoiceFlow v1.2)

8 инструментов, вызываются агентом через LangChain `@tool`.

---

## Режим A — Пациент (7 инструментов)

### 1. `search_knowledge_base`
**Поиск по базе знаний клиники.** Цены, услуги, врачи, адрес, часы работы, FAQ.

| Параметр | Тип | Описание |
|---|---|---|
| `query` | `str` | Поисковый запрос |

**Правило:** вызывать ПЕРЕД любым ответом о фактической информации. Никогда не отвечать по памяти.

---

### 2. `check_calendar_slots`
**Свободные слоты из Google Calendar.**

| Параметр | Тип | Обязательный | Описание |
|---|---|---|---|
| `date_from` | `str` | да | Дата начала (YYYY-MM-DD) |
| `date_to` | `str` | да | Дата окончания (YYYY-MM-DD) |
| `doctor` | `str` | нет | Тип врача: `therapist` / `surgeon` |
| `service` | `str` | нет | Тип услуги: `consultation` / `cleaning` / `treatment` |

**Правило:** вызывать ПЕРЕД тем, как предложить пациенту время.

---

### 3. `create_appointment`
**Создать запись в БД + Google Calendar.**

| Параметр | Тип | Описание |
|---|---|---|
| `patient_name` | `str` | Имя пациента |
| `phone` | `str` | Номер телефона |
| `service` | `str` | Тип услуги: `consultation` / `cleaning` / `treatment` |
| `datetime_str` | `str` | Дата и время (YYYY-MM-DDTHH:MM) |
| `doctor` | `str` | Тип врача: `therapist` / `surgeon` |

**Длительность:** consultation — 30 мин, cleaning — 60 мин, treatment — 90 мин.

**Правило:** вызывать ТОЛЬКО после явного согласия пациента и получения имени+телефона.

---

### 4. `update_appointment`
**Перенести запись.**

| Параметр | Тип | Обязательный | Описание |
|---|---|---|---|
| `appointment_id` | `int` | да | ID записи |
| `new_datetime` | `str` | нет | Новая дата/время (YYYY-MM-DDTHH:MM) |
| `new_doctor` | `str` | нет | Новый тип врача |

---

### 5. `cancel_appointment`
**Отменить запись.**

| Параметр | Тип | Обязательный | Описание |
|---|---|---|---|
| `appointment_id` | `int` | да | ID записи |
| `reason` | `str` | нет | Причина отмены |

---

### 6. `view_appointments`
**Показать активные записи пользователя.** Без параметров.

---

### 7. `escalate_to_admin`
**Эскалация администратору клиники.**

| Параметр | Тип | Обязательный | Описание |
|---|---|---|---|
| `summary` | `str` | да | Краткое описание ситуации |
| `patient_contact` | `str` | да | Контакт пациента |
| `urgency` | `str` | нет | Срочность: `low` / `normal` / `high` (default: `normal`) |

**Триггеры:** жалобы на качество/врача, возврат средств, заказ такси, заказ лекарств, юридические вопросы, запрос медкарты, просьба соединить с человеком, услуга/цена не найдена после двух поисков.

---

## Режим B — Собственник (1 инструмент)

### 8. `notify_sales`
**Оповещение в чат администраторов AI Soft Retail.**

| Параметр | Тип | Обязательный | Описание |
|---|---|---|---|
| `stage` | `str` | да | `warm` / `hot` |
| `owner_name` | `str` | нет | Имя собственника (для hot) |
| `clinic_name` | `str` | нет | Название клиники |
| `owner_contact` | `str` | нет | Контакт: телефон или @username |
| `locations` | `int` | нет | Количество филиалов |
| `details` | `str` | нет | Резюме диалога или детали |

**Правила вызова:**
- `warm` — собственник проявил интерес (назвал клинику). Отправляется ссылка `https://t.me/<username>` и краткое резюме.
- `hot` — собственник готов купить. Отправляется имя, телефон, клиника, филиалы.

---

## Инъекция зависимостей

Все инструменты получают runtime-зависимости через `RunnableConfig` (`configurable`):
- `session` — SQLAlchemy `AsyncSession`
- `user` — объект `User` (текущий пользователь Telegram)
- `admin_bot` — экземпляр Telegram-бота для отправки уведомлений
- `calendar_service` — `GoogleCalendarService` (опционально)
- `language` — язык диалога (`ru`/`uz`/`en`)
- `thread_id` — ID чата для MemorySaver
