# Инструменты агента Мадины (VoiceFlow v1.2)

13 инструментов, вызываются агентом через LangChain `@tool`.

---

## Режим A — Пациент (9 инструментов)

### 1. `search_knowledge_base`
**Поиск по базе знаний клиники.** Цены, услуги, врачи, адрес, часы работы, FAQ.

| Параметр | Тип | Описание |
|---|---|---|
| `query` | `str` | Поисковый запрос |

**Правило:** вызывать ПЕРЕД любым ответом о фактической информации. Никогда не отвечать по памяти.

---

### 2. `check_calendar_slots`
**Свободные слоты из Google Calendar.**

| Параметр | Тип | Обязательный | Описание |
|---|---|---|---|
| `date_from` | `str` | да | Дата начала (YYYY-MM-DD) |
| `date_to` | `str` | да | Дата окончания (YYYY-MM-DD) |
| `doctor` | `str` | нет | Тип врача: `therapist` / `surgeon` |
| `service` | `str` | нет | Тип услуги: `consultation` / `cleaning` / `treatment` |

**Правило:** вызывать ПЕРЕД тем, как предложить пациенту время.

---

### 3. `create_appointment`
**Создать запись в БД + Google Calendar.**

| Параметр | Тип | Описание |
|---|---|---|
| `patient_name` | `str` | Имя пациента |
| `phone` | `str` | Номер телефона |
| `service` | `str` | Тип услуги |
| `datetime_str` | `str` | Дата и время (YYYY-MM-DDTHH:MM) |
| `doctor` | `str` | Тип врача (опционально) |

**Правило:** вызывать ТОЛЬКО после явного согласия пациента и получения имени+телефона.

---

### 4. `update_appointment`
**Перенести запись.**

| Параметр | Тип | Обязательный | Описание |
|---|---|---|---|
| `appointment_id` | `int` | да | ID записи |
| `new_datetime` | `str` | нет | Новая дата/время (YYYY-MM-DDTHH:MM) |
| `new_doctor` | `str` | нет | Новый тип врача |

---

### 5. `cancel_appointment`
**Отменить запись.**

| Параметр | Тип | Обязательный | Описание |
|---|---|---|---|
| `appointment_id` | `int` | да | ID записи |
| `reason` | `str` | нет | Причина отмены |

---

### 6. `view_appointments`
**Показать активные записи пользователя.** Без параметров.

---

### 7. `order_taxi`
**Заказ такси.** Плейсхолдер — создаёт escalation для ручной обработки.

| Параметр | Тип | Обязательный | Описание |
|---|---|---|---|
| `pickup` | `str` | да | Адрес подачи |
| `destination` | `str` | да | Адрес назначения |
| `when` | `str` | нет | Когда подать |

**Правило:** только по явной просьбе пациента или при явном поводе (пожилой, удаление зуба, далеко живёт).

---

### 8. `order_pharmacy`
**Заказ лекарств.** Плейсхолдер — создаёт escalation.

| Параметр | Тип | Обязательный | Описание |
|---|---|---|---|
| `items` | `str` | да | Список лекарств через запятую |
| `address` | `str` | да | Адрес доставки |
| `when` | `str` | нет | Когда доставить |

**Правило:** только по явной просьбе.

---

### 9. `escalate_to_admin`
**Эскалация администратору клиники.**

| Параметр | Тип | Обязательный | Описание |
|---|---|---|---|
| `summary` | `str` | да | Краткое описание ситуации |
| `patient_contact` | `str` | да | Контакт пациента |
| `urgency` | `str` | нет | Срочность: `low` / `normal` / `high` (default: `normal`) |

**Триггеры:** жалобы на качество/врача, возврат средств, юридические вопросы, запрос медкарты, просьба соединить с человеком, услуга/цена не найдена после двух поисков.

---

## Режим B — Собственник (4 инструмента)

### 10. `notify_sales`
**Алерт в Telegram-канал «Заказы от Мадины».**

| Параметр | Тип | Обязательный | Описание |
|---|---|---|---|
| `stage` | `str` | да | `warm` / `hot` / `cold_lead` |
| `clinic_name` | `str` | нет | Название клиники |
| `owner_contact` | `str` | да | Контакт собственника |
| `details` | `str` | нет | Детали (имя, локации, сумма) |

**Правила вызова:**
- `warm` — собственник назвал клинику (сразу)
- `hot` — собственник согласился подключиться
- `cold_lead` — собственник не ответил после возвратного сообщения

---

### 11. `handoff_to_amir`
**Передача коллеге Амиру для консультации.**

| Параметр | Тип | Обязательный | Описание |
|---|---|---|---|
| `clinic_name` | `str` | нет | Название клиники |
| `owner_contact` | `str` | да | Контакт собственника |
| `conversation_summary` | `str` | да | Краткое резюме разговора |
| `development_zone` | `str` | да | Зона интереса |

**Значения `development_zone`:**
- `data_privacy` — копает в безопасность данных глубже базовой фразы
- `general_ai_interest` — интересуется другими AI-продуктами
- `hesitating` — колеблется, не готов к VoiceFlow
- `post_purchase` — после закрытия сделки (опциональный бонус)

**После вызова** — сообщить контакт Амира: `@Ai_Soft_Retail_a8i_bot`

---

### 12. `schedule_followup`
**Запланировать возврат к разговору.**

| Параметр | Тип | Обязательный | Описание |
|---|---|---|---|
| `owner_contact` | `str` | да | Контакт собственника |
| `when_iso` | `str` | да | Дата/время возврата (ISO: YYYY-MM-DDTHH:MM:SS+TZ) |
| `context_summary` | `str` | да | О чём говорили |
| `timezone` | `str` | нет | Часовой пояс (default: `Asia/Tashkent`) |

**Правило:** только ОДИН возврат. Если собственник не ответил — `notify_sales(stage="cold_lead")`.

---

### 13. `send_invoice`
**Отправить счёт на оплату.** Плейсхолдер — создаёт escalation для ручной отправки Иваном.

| Параметр | Тип | Обязательный | Описание |
|---|---|---|---|
| `owner_name` | `str` | да | Имя собственника |
| `clinic_name` | `str` | да | Название клиники |
| `phone` | `str` | да | Номер телефона |
| `amount` | `str` | да | Сумма (например `$200/мес`) |
| `locations` | `int` | нет | Количество локаций (default: 1) |

**Правило:** вызывать после `notify_sales(stage="hot")` при закрытии сделки.

---

## Инъекция зависимостей

Все инструменты получают runtime-зависимости через `RunnableConfig` (`configurable`):
- `session` — SQLAlchemy `AsyncSession`
- `user` — объект `User` (текущий пользователь Telegram)
- `admin_bot` — экземпляр Telegram-бота для отправки уведомлений
- `calendar_service` — `GoogleCalendarService` (опционально)
- `language` — язык диалога (`ru`/`uz`/`en`)
- `thread_id` — ID чата для MemorySaver
